## Changes

### Bug fixes (training performance regression)

- **pose/mmpose/engine/optim_wrappers/fixed_amp_optim_wrapper.py** (new): Created `FixedAmpOptimWrapper` that subclasses MMEngine's `AmpOptimWrapper` to work around a bug in the Sapiens fork. The upstream `amp_optimizer_wrapper.py` zeros out inf/nan gradients *before* `loss_scaler.unscale_()`, which prevents the GradScaler from detecting overflow — the scaler never reduces its loss scale, so every subsequent iteration overflows silently with zero gradients. The fix restores the standard PyTorch AMP flow (unscale → clip → step → update).

- **pose/mmpose/engine/optim_wrappers/__init__.py**: Added export for `FixedAmpOptimWrapper`.

- **pose/tools/train.py**: Changed AMP wrapper from `AmpOptimWrapper` to `FixedAmpOptimWrapper`.

- **pose/configs/sapiens_pose/bedlam2/sapiens_0.3b-50e_bedlam2-640x384.py**: (1) Added `fixed_amp_optim_wrapper` to `custom_imports`. (2) Changed warmup `start_factor` from 0.001 to 0.333 to match standalone LR schedule (epoch 1 head LR was 1e-7 instead of 3.33e-5). (3) Fixed stale `--amp` comment.

### Transformer decoder head implementation (Issues 1 & 2)

- **pose/mmpose/models/heads/regression_heads/pose3d_transformer_head.py** (new): `Pose3dTransformerHead` — transformer decoder head with 2D sinusoidal pos enc, 70 learnable joint queries, 1 decoder layer (self-attn → cross-attn → FFN), single linear output projections, pelvis branches from token 0. Drop-in replacement for `Pose3dRegressionHead`.

- **pose/tests/test_models/test_heads/test_pose3d_transformer_head.py** (new): 13 unit tests covering forward shapes, loss computation, predict format, and embed_dim generalization. All passing.

- **Smoke test passed**: config loads, model builds (16.9M head params), training runs and converges (loss 2.45 → 1.06, MPJPE 538 → 335mm over 850 iterations of epoch 1).
