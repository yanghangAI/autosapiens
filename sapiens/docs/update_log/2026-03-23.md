## Summary

Reduced max depth from 20 m to 10 m; added far-person and OOB filters for both train and val.

## Changed Files

- `pose/mmpose/datasets/transforms/bedlam2_transforms.py`
  - `_DEPTH_MAX_METERS`: 20.0 → 10.0 (depth normalization ceiling)
  - `LoadBedlamLabels.transform()`: added filter — skips samples (train only, `filter_invalid=True`) where all 70 active joints have X (depth) ≥ 10 m; returns `None` to trigger MMEngine retry
  - Updated docstring to document the new far-person filter

- `pose/mmpose/datasets/datasets/body3d/bedlam2_dataset.py`
  - `load_data_list()`: pre-filters samples where all active joints have X ≥ `_DEPTH_MAX_METERS` (far-person) and where ≥ 50% of 127 joints are OOB (infers W/H from intrinsic principal point) — covers val/test where transform-level `None` returns are disallowed
  - Imports `_ACTIVE_JOINT_INDICES` and `_DEPTH_MAX_METERS` from `bedlam2_transforms`

- `pose/mmpose/datasets/transforms/bedlam2_transforms.py` (additional)
  - OOB threshold: `> 0.70` → `>= 0.50`

- `pose/configs/sapiens_pose/bedlam2/sapiens_0.3b-50e_bedlam2-640x384-transformer.py`
- `pose/configs/sapiens_pose/bedlam2/sapiens_0.3b-50e_bedlam2-640x384.py`
  - Fixed default `data_root`: `BEDLAM2Datatest` → `BEDLAM2Datatest/data` (was off by one `data/` level)
