# 2026-03-20

## Summary
Fixed stale `mpjpe`/`mpjpe_abs` terminal scalars on checkpoint resume. Two bugs: (1) purge was in `before_run` (fires before checkpoint load); (2) wrong key names — MMEngine stores loss keys as `train/{key}`, not bare `{key}`. Also cleaned up run dirs (removed 12 startup-only log dirs) and documented training results.

## Changes

### `pose/mmpose/engine/hooks/train_mpjpe_hook.py`
- **Hook timing fix:** Renamed `before_run` → `before_train`. `before_run` fires before `load_or_resume()`, so checkpoint restored stale keys after the purge. `before_train` fires after `load_or_resume()`.
- **Key name fix:** Changed purge keys from `'mpjpe'`/`'mpjpe_abs'` to `'train/mpjpe'`/`'train/mpjpe_abs'`. `RuntimeInfoHook.after_train_iter` stores all `outputs` keys as `f'train/{key}'` in `message_hub.log_scalars`, so the stale checkpoint keys have the `train/` prefix.
- Also purges `_resumed_keys` to prevent stale keys being re-saved into future checkpoints.
