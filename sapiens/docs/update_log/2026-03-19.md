# 2026-03-19

## Summary
Fixed three bugs in `TrainMPJPEAveragingHook`: DDP wrapper traversal, TensorBoard-only logging, and stale MessageHub keys from resumed checkpoints causing flat per-iteration `mpjpe`/`mpjpe_abs` in terminal.

## Changes

### `pose/mmpose/engine/hooks/train_mpjpe_hook.py`
- **DDP wrapper fix:** `after_train_iter` now unwraps `runner.model.module` if present before reading `head._train_mpjpe`/`_train_mpjpe_abs`. Previously `runner.model.head` was always `None` under DDP, so the buffer never accumulated any values.
- **Logging backend fix:** `after_train_epoch` now uses `runner.visualizer.add_scalar` instead of writing directly to `tb._tensorboard`. Values now propagate to all backends (TensorBoard, `scalars.json`, terminal).
- **Stale key purge:** Added `before_run` that removes `mpjpe` and `mpjpe_abs` from `runner.message_hub.log_scalars`. Older checkpoints wrote these directly into the losses dict; on resume they reappeared as frozen per-iteration scalars in the terminal.
- Removed now-unused `_get_tb_writer` static method and `Sequence` import.
