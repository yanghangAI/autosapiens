# 2026-03-18

## Summary
1. Fixed garbled skeleton visualization caused by wrong SMPL-X joint ordering in body link definitions. Added skeleton link drawing to demo script.
2. Restructured TensorBoard logging: unified train/val MPJPE tags, added absolute MPJPE metric, epoch-averaged training MPJPE hook.
3. Fixed two TensorBoard tag bugs found by verifying against `bedlam2_transformer_0.3b/20260318_152401`: (a) `mpjpe/abs/val` was never computed because `K` was looked up in `metainfo` but MMEngine's `to_dict()` flattens metainfo to the top level; (b) raw `mpjpe`/`mpjpe_abs` per-batch tags were being auto-logged by MMEngine — moved to head attributes, hook reads them directly.
3. Restructured docs in both `sapiens/docs/` and `pose/docs/`.

### Bug fix: wrong SMPL-X body joint ordering in skeleton links

- **pose/mmpose/engine/hooks/pose3d_visualization_hook.py**: Replaced `_BODY_LINKS` — the old links assumed a non-standard joint ordering (hips→knees→ankles→spine→...) but BEDLAM2 uses standard SMPL-X ordering (0=pelvis, 1=L_hip, 2=R_hip, 3=spine1, 4=L_knee, 5=R_knee, 6=spine2, ...). This caused garbled skeleton lines in TensorBoard videos (e.g., pelvis→left_ankle instead of pelvis→spine1). Corrected to match `claude_code/data/constants.py::_SMPLX_BONES_RAW`.

- **pose/demo/demo_bedlam2.py**: Same fix to `BODY_LINKS`. Also added skeleton link drawing to `_draw_skeleton_2d` (was dots-only) and `_draw_skeleton_3d` (was scatter-only).

### Bug fix: TensorBoard tag issues (verified against 20260318_152401 run)

- **pose/mmpose/evaluation/metrics/bedlam_metric.py**: Fixed `mpjpe/abs/val` never being computed. Root cause: `process()` looked for `K` inside `data_sample['metainfo']`, but MMEngine's `Evaluator.process()` calls `to_dict()` which flattens metainfo to the top level of the dict. Changed condition to check `'K' in data_sample` and access `data_sample['K']` / `data_sample.get('img_shape', ...)` directly.
- **pose/mmpose/models/heads/regression_heads/pose3d_regression_head.py**: Removed `losses['mpjpe']` and `losses['mpjpe_abs']` from the losses dict. MMEngine auto-logs everything in the losses dict, producing noisy per-batch TensorBoard scalars. Values now stored as `self._train_mpjpe` / `self._train_mpjpe_abs` instance attributes for the hook to read.
- **pose/mmpose/models/heads/regression_heads/pose3d_transformer_head.py**: Same change as regression head.
- **pose/mmpose/engine/hooks/train_mpjpe_hook.py**: Updated `after_train_iter` to read `runner.model.head._train_mpjpe` / `._train_mpjpe_abs` instead of `outputs['mpjpe']` / `outputs['mpjpe_abs']`.
- **pose/tests/test_tensorboard_restructure.py**: Updated `EXPECTED_KEYS` (removed `mpjpe`/`mpjpe_abs`), `TestHeadMpjpeAbs` (checks head attributes, not losses), and `TestTrainMPJPEHook` (uses fake head with attributes).
